# Nuxt 대시보드 프로젝트 시리즈 (3): Pinia를 이용한 상태 관리 설계

대시보드에서는 사이드바의 접힘 상태나 현재 로그인한 사용자의 정보가 여러 컴포넌트에서 공유되어야 합니다. Pinia를 사용하여 깔끔하고 타입 안전한 상태 관리 환경을 구축해 보겠습니다.

---

## 1. 전역 UI 상태 관리 (App Store)

사이드바의 상태를 관리하는 간단한 스토어를 만듭니다.

```typescript
// stores/app.ts
import { defineStore } from 'pinia'

export const useAppStore = defineStore('app', () => {
  // 상태 (State)
  const isSidebarCollapsed = ref(false)
  const theme = ref<'light' | 'dark'>('light')

  // 액션 (Actions)
  function toggleSidebar() {
    isSidebarCollapsed.value = !isSidebarCollapsed.value
  }

  function toggleTheme() {
    theme.value = theme.value === 'light' ? 'dark' : 'light'
  }

  return { isSidebarCollapsed, theme, toggleSidebar, toggleTheme }
})
```

---

## 2. 사용자 인증 정보 관리 (Auth Store)

사용자의 프로필 정보를 담는 스토어입니다.

```typescript
// stores/auth.ts
export const useAuthStore = defineStore('auth', () => {
  const user = ref({
    name: '관리자',
    role: 'ADMIN',
    avatar: 'https://api.dicebear.com/7.x/avataaars/svg?seed=Admin'
  })

  const isLoggedIn = computed(() => !!user.value)

  return { user, isLoggedIn }
})
```

---

## 3. 컴포넌트에서 사용하기

Setup 문법을 사용하여 직관적으로 상태를 가져옵니다.

```vue
<script setup>
import { useAppStore } from '@/stores/app'

const appStore = useAppStore()
</script>

<template>
  <button @click="appStore.toggleSidebar">
    {{ appStore.isSidebarCollapsed ? '열기' : '닫기' }}
  </button>
</template>
```

---

## 4. Pinia 설계 팁

1.  **모듈화:** 하나의 큰 스토어보다는 `app`, `auth`, `project` 등 관심사별로 스토어를 나누세요.
2.  **SSR 고려:** Nuxt에서 Pinia를 쓸 때는 서버에서도 상태가 유지되도록 `@pinia/nuxt` 설정을 확인해야 합니다.
3.  **Persistence:** 상태를 새로고침 후에도 유지하고 싶다면 `pinia-plugin-persistedstate` 라이브러리를 추가해 보세요.
