# HTTP와 HTTPS 이해하기: 웹 통신의 근간과 보안

우리가 매일 사용하는 웹 브라우저 주소창의 시작은 항상 `http://` 또는 `https://`입니다. 이 두 프로토콜이 어떻게 데이터를 주고받는지, 그리고 왜 보안(SSL/TLS)이 필수인지 정리합니다.

---

## 1. HTTP (HyperText Transfer Protocol)

HTTP는 클라이언트(브라우저)와 서버가 텍스트 기반의 데이터를 주고받기 위한 약속입니다.

*   **상태 없음 (Stateless):** 서버는 클라이언트의 이전 요청을 기억하지 않습니다. (이를 보완하기 위해 쿠키와 세션을 사용합니다.)
*   **비연결성 (Connectionless):** 요청에 대한 응답을 보내면 연결을 끊어버려 서버 자원을 효율적으로 관리합니다.
*   **취약점:** 데이터가 평문(Plain Text)으로 전송되므로, 중간에 누군가 데이터를 가로채면 내용을 그대로 볼 수 있습니다.

---

## 2. HTTPS (HTTP Secure)

HTTPS는 HTTP에 **보안(SSL/TLS)** 계층을 추가한 프로토콜입니다. 모든 데이터를 암호화하여 전송합니다.

### **작동 방식 (SSL Handshake)**
1.  **공개키 암호화:** 데이터를 암호화할 때 쓰는 키와 복호화할 때 쓰는 키가 다릅니다. (보안성 높음)
2.  **대칭키 암호화:** 암호화와 복호화에 같은 키를 씁니다. (속도 빠름)
3.  **HTTPS의 지혜:** 처음 연결할 때만 공개키 방식으로 안전하게 '대칭키'를 공유하고, 이후 데이터 전송은 빠른 대칭키 방식을 사용합니다.

---

## 3. 왜 HTTPS를 써야 할까?

1.  **기밀성:** 데이터가 암호화되어 해킹으로부터 안전합니다.
2.  **무결성:** 데이터가 중간에 변조되지 않았음을 보장합니다.
3.  **인증:** 접속한 서버가 신뢰할 수 있는 진짜 서버인지 확인할 수 있습니다.
4.  **SEO 혜택:** 구글 등 검색 엔진은 HTTPS를 사용하는 사이트에 가산점을 부여합니다.

---

## 4. 요약

*   **HTTP:** 빠르지만 보안이 없습니다.
*   **HTTPS:** 암호화 과정으로 인해 HTTP보다 아주 미세하게 느릴 수 있지만, 현대 웹에서는 **필수**입니다.

보안은 타협의 대상이 아닙니다. 2026년 현재, 모든 웹 서비스는 HTTPS를 기본으로 운영되어야 합니다.
