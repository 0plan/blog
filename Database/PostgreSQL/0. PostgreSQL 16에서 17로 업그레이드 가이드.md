# PostgreSQL 16에서 17로 업그레이드 가이드: 주요 변경점 및 마이그레이션

PostgreSQL 17은 성능 최적화, 관리 편의성 향상, 그리고 SQL 표준 지원 강화에 중점을 둔 대규모 업데이트입니다. 특히 진보된 진공(VACUUM) 프로세스와 증분 백업(Incremental Backup) 기능은 데이터베이스 운영 효율성을 획기적으로 높여줍니다.

## 1. PostgreSQL 17 주요 변경점

### 1.1 성능 및 운영 최적화
- **VACUUM 메모리 관리 혁신**: VACUUM 작업 시 사용하는 메모리 관리 방식이 완전히 재설계되어, 메모리 소비량이 이전 버전 대비 최대 **20배** 감소했습니다. 이는 고부하 시스템에서 자원 경합을 줄여줍니다.
- **스트리밍 I/O 도입**: 순차 스캔(Sequential Scan) 및 `ANALYZE` 작업 속도를 높여주는 새로운 스트리밍 I/O 인터페이스가 추가되었습니다.
- **증분 백업(Incremental Backup)**: `pg_basebackup`에서 증분 백업을 기본 지원합니다. 또한 `pg_combinebackup` 도구를 사용하여 증분 백업본을 전체 백업본으로 복원할 수 있게 되어 백업 효율성이 극대화되었습니다.

### 1.2 SQL 및 개발자 편의 기능
- **SQL/JSON 표준 지원**: 
    - `JSON_TABLE()` 함수를 통해 JSON 데이터를 관계형 테이블 형태로 변환하여 SQL 조인이나 필터링에 활용할 수 있습니다.
    - `JSON_EXISTS()`, `JSON_QUERY()`, `JSON_VALUE()` 등 새로운 쿼리 함수들이 추가되었습니다.
- **MERGE 구문 강화**: `MERGE` 명령에서 `RETURNING` 절을 사용할 수 있게 되었으며, 업데이트 가능한 뷰(Updatable Views)를 대상으로도 실행할 수 있습니다.
- **COPY 오류 무시**: `COPY` 명령어에 `ON_ERROR ignore` 옵션이 추가되어, 일부 행에 에러가 있더라도 작업을 중단하지 않고 계속 진행할 수 있습니다.

### 1.3 논리적 복제(Logical Replication)
- **pg_createsubscriber**: 물리적 대기 서버(Physical Standby)를 논리적 복제본(Logical Replica)으로 손쉽게 전환할 수 있는 새로운 유틸리티가 도입되었습니다.
- **장애 조치(Failover) 지원**: 기본 서버 장애 시에도 구독자(Subscriber)가 동기화 상태를 유지할 수 있도록 제어 기능이 개선되었습니다.

## 2. 업그레이드 전 주의사항

- **논리적 복제 슬롯**: PostgreSQL 17로 업그레이드할 때는 기존의 논리적 복제 슬롯이 보존되지 않습니다(17에서 18로 갈 때는 가능). 따라서 업그레이드 전 슬롯을 삭제하고 업그레이드 후 다시 생성해야 합니다.
- **Search Path 보안**: 보안 강화를 위해 관리 작업 중 `search_path` 규칙이 더 엄격하게 적용됩니다. 인덱스 식이나 구체화된 뷰에서 사용되는 함수는 스키마 명칭을 명시적으로 지정하는 것이 권장됩니다.
- **지원 중단**: AIX OS 지원이 공식적으로 중단되었습니다.

## 3. 마이그레이션 절차 (pg_upgrade)

가장 권장되는 방식은 데이터 파일을 직접 복사하지 않고 링크만 생성하는 `--link` 모드를 사용하는 것입니다.

```bash
# 1. 새 버전(17)의 바이너리 설치 및 데이터 디렉토리 초기화(initdb)
# 2. 업그레이드 호환성 체크
pg_upgrade \
  --old-datadir /var/lib/postgresql/16/main \
  --new-datadir /var/lib/postgresql/17/main \
  --old-bindir /usr/lib/postgresql/16/bin \
  --new-bindir /usr/lib/postgresql/17/bin \
  --check

# 3. 실제 업그레이드 수행 (하드링크 모드 권장)
pg_upgrade \
  --old-datadir /var/lib/postgresql/16/main \
  --new-datadir /var/lib/postgresql/17/main \
  --old-bindir /usr/lib/postgresql/16/bin \
  --new-bindir /usr/lib/postgresql/17/bin \
  --link
```

## 4. 업그레이드 후 마무리 작업

- **통계 정보 갱신**: 업그레이드 직후에는 실행 계획 최적화를 위해 `./analyze_new_cluster.sh`를 실행하여 통계 정보를 다시 수집해야 합니다.
- **익스텐션 업데이트**: PostGIS 등 사용 중인 외부 확장이 17 버전에 맞게 업데이트되었는지 확인하고 필요한 경우 재설치합니다.
- **인덱스 재구성**: 정렬(Collation) 관련 라이브러리 업데이트가 있었다면 `REINDEX`를 통해 인덱스 무결성을 확인하는 것이 좋습니다.

---
PostgreSQL 17은 특히 대규모 데이터를 다루는 환경에서 VACUUM 성능 향상과 증분 백업만으로도 충분히 업그레이드할 가치가 있는 버전입니다.
