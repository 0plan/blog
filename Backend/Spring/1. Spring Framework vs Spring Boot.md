# Spring Framework vs Spring Boot: 그 관계와 차이점

자바 백엔드 개발자에게 **Spring(스프링)**은 떼려야 뗄 수 없는 존재입니다. 하지만 처음 시작하는 분들에게는 "스프링 프레임워크"와 "스프링 부트"가 각각 무엇인지, 왜 요즘은 모두가 스프링 부트를 쓰는지 혼란스러울 수 있습니다. 2026년 현재의 관점에서 이 둘의 관계를 명확히 정리해 보겠습니다.

---

## 1. Spring Framework: 거대한 기반 기술

**Spring Framework**는 자바 엔터프라이즈 애플리케이션 개발을 위한 종합적인 인프라를 제공합니다. 핵심은 **IoC(제어의 역전)**와 **DI(의존성 주입)**를 통해 객체 간의 결합도를 낮추는 것입니다.

*   **주요 기능:** 핵심 컨테이너, 데이터 액세스(JDBC, ORM), 웹(MVC), AOP(관점 지향 프로그래밍) 등.
*   **특징:** 매우 강력하고 유연하지만, 설정이 매우 복잡합니다. 수많은 XML 설정이나 복잡한 Java Config가 필요하며, 톰캣(Tomcat) 같은 웹 서버를 별도로 설치하고 WAR 파일을 배포해야 했습니다.

---

## 2. Spring Boot: 스프링을 "그냥 사용하게" 해주는 도구

**Spring Boot**는 스프링 프레임워크를 기반으로, 개발자가 설정을 최소화하고 바로 비즈니스 로직에 집중할 수 있도록 해주는 **'의견이 있는(Opinionated)'** 프레임워크입니다.

### **스프링 부트의 3대 핵심 특징**
1. **Auto Configuration (자동 설정):** 클래스패스에 있는 라이브러리를 보고 필요한 빈(Bean)들을 자동으로 설정해 줍니다. (예: `spring-boot-starter-web`이 있으면 자동으로 톰캣을 설정함)
2. **Starter Dependencies (스타터 의존성):** 관련된 라이브러리들을 묶음으로 제공하여 복잡한 의존성 관리 문제를 해결합니다.
3. **Embedded Server (내장 서버):** 톰캣 같은 웹 서버가 라이브러리 형태로 내장되어 있어, 별도의 서버 설치 없이 `java -jar` 명령만으로 실행이 가능합니다.

---

## 3. 2026년 최신 트렌드: Spring Boot 3.x

현재는 **Spring Boot 3.x (Spring Framework 6.x)**가 표준입니다. 이전 버전과 비교하여 다음과 같은 혁신적인 변화가 있습니다.

*   **Virtual Threads (Project Loom) 지원:** 기존의 무거운 OS 스레드 대신 수백만 개의 가상 스레드를 활용하여 동시성 처리가 획기적으로 향상되었습니다.
*   **GraalVM Native Image:** 애플리케이션을 네이티브 바이너리로 컴파일하여 시작 시간(Startup time)을 밀리초(ms) 단위로 단축하고 메모리 사용량을 절반 이하로 줄였습니다. (클라우드 네이티브 환경에 최적)
*   **Jakarta EE 11 지원:** 최신 자바 표준 사양을 완벽히 따릅니다.

---

## 4. 요약: 무엇을 써야 할까?

| 비교 항목 | Spring Framework | Spring Boot |
| :--- | :--- | :--- |
| **설정 방식** | 명시적이고 상세한 설정 필요 | **자동 설정 (Auto-config)** |
| **배포 방식** | 외장 WAS 설치 및 WAR 배포 | **내장 서버를 통한 단독 JAR 실행** |
| **생산성** | 초기 설정에 많은 시간 소요 | **프로젝트 생성 즉시 비즈니스 개발 가능** |
| **추천 프로젝트** | 특수한 레거시 시스템 유지보수 | **대부분의 신규 프로젝트, 마이크로서비스** |

### **결론**
Spring Boot는 Spring Framework의 대안이 아니라, **Spring Framework를 더 쉽고 강력하게 사용할 수 있도록 감싸고 있는 도구**입니다. 오늘날 자바 백엔드 개발을 시작한다면 고민할 것 없이 **Spring Boot**로 시작하시면 됩니다.
