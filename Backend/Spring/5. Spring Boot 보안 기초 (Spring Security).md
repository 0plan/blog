# Spring Boot 보안 기초 (Spring Security)

스프링 부트 애플리케이션의 인증(Authentication)과 인가(Authorization)를 담당하는 핵심 보안 프레임워크인 **Spring Security**의 기초 설정과 활용법을 정리합니다. 2026년 기준 최신 보안 설정을 반영합니다.

---

## 1. Spring Security 개요

Spring Security는 "인증(누구인가?)"과 "인가(어떤 권한이 있는가?)"를 제어하는 강력한 보안 프레임워크입니다. **필터 기반**으로 작동하며, 스프링 부트와 결합하여 대부분의 설정을 자동으로 수행해 줍니다.

---

## 2. 기본 설정 (Dependencies)

### **Maven (`pom.xml`)**
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

### **Gradle (`build.gradle`)**
```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-security'
}
```

## 3. 보안 설정 커스터마이징 (SecurityConfig)

최신 스프링 시큐리티는 `WebSecurityConfigurerAdapter` 상속 방식이 아닌, **`SecurityFilterChain`** 빈을 등록하는 방식을 사용합니다.

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable()) // REST API 환경에서는 CSRF 비활성화 권장
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll() // 누구나 접근 가능
                .requestMatchers("/api/admin/**").hasRole("ADMIN") // 관리자만 접근 가능
                .anyRequest().authenticated() // 나머지는 로그인 필수
            )
            .httpBasic(Customizer.withDefaults()); // 기본 인증 방식 사용
            
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(); // 비밀번호는 반드시 암호화하여 저장
    }
}
```

---

## 4. 사용자 인증 (InMemoryUser)

테스트 목적으로 메모리에 사용자를 등록하는 방법입니다. 실무에서는 DB 연동(`UserDetailsService`)을 사용합니다.

```java
@Bean
public UserDetailsService userDetailsService() {
    UserDetails user = User.builder()
            .username("user")
            .password(passwordEncoder().encode("1234"))
            .roles("USER")
            .build();
    return new InMemoryUserDetailsManager(user);
}
```

---

## 5. 핵심 보안 개념

1.  **CSRF (Cross-Site Request Forgery):** 사이트 간 요청 위조 방지 기능. REST API(State-less) 환경에서는 주로 끄고 토큰(JWT 등) 방식을 씁니다.
2.  **CORS (Cross-Origin Resource Sharing):** 다른 도메인(예: React 앱)에서 백엔드 API를 호출할 수 있도록 허용하는 설정입니다.
3.  **AuthenticationManager:** 실제 인증 로직을 총괄하는 핵심 컴포넌트입니다.

---

## 6. 요약

*   의존성 추가만으로 기본적인 보안이 시작됩니다.
*   `SecurityFilterChain`을 통해 세밀한 접근 권한(인가)을 설정합니다.
*   **비밀번호 암호화**는 선택이 아닌 필수입니다 (`BCrypt`).

보안은 기능을 완성한 후에 덧붙이는 것이 아니라, 설계 단계부터 고려해야 하는 필수 요소입니다.
