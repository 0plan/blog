# Spring Boot 데이터베이스 연동: Spring Data JPA 활용하기

자바 백엔드 개발에서 데이터베이스 연동은 가장 핵심적인 부분입니다. **Spring Data JPA**를 사용하면 복잡한 SQL 쿼리 작성 없이도 객체 지향적으로 데이터를 관리할 수 있습니다. 2026년 기준 실무에서 가장 권장되는 연동 방식을 정리합니다.

---

## 1. 의존성 추가 (Dependencies)

### **Maven (`pom.xml`)**
```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-jpa</artifactId>
</dependency>
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

### **Gradle (`build.gradle`)**
```groovy
dependencies {
    implementation 'org.springframework.boot:spring-boot-starter-data-jpa'
    runtimeOnly 'com.h2database:h2'
}
```

---

## 2. 데이터베이스 설정 (application.properties)

DB 접속 정보를 설정 파일에 기입합니다.

```properties
# H2 인메모리 DB 설정 예시
spring.datasource.url=jdbc:h2:mem:testdb
spring.datasource.driverClassName=org.h2.Driver
spring.datasource.username=sa
spring.datasource.password=

# JPA 상세 설정
spring.jpa.hibernate.ddl-auto=update
spring.jpa.show-sql=true
spring.jpa.properties.hibernate.format_sql=true
```

---

## 3. 핵심 구성 요소

### **① 엔티티 (Entity)**
데이터베이스 테이블과 매핑될 자바 클래스입니다.

```java
@Entity
@Getter @Setter
public class User {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false)
    private String name;
    
    private String email;
}
```

### **② 리포지토리 (Repository)**
인터페이스 선언만으로 기본적인 CRUD 기능을 제공받습니다.

```java
public interface UserRepository extends JpaRepository<User, Long> {
    // 메소드 이름만으로 쿼리 생성 가능
    Optional<User> findByEmail(String email);
}
```

---

## 4. 서비스 레이어에서의 사용

비즈니스 로직은 `@Service` 클래스에서 구현하며, `@Transactional` 어노테이션을 통해 데이터의 일관성을 보장합니다.

```java
@Service
@RequiredArgsConstructor
public class UserService {
    private final UserRepository userRepository;

    @Transactional
    public Long join(User user) {
        userRepository.save(user);
        return user.getId();
    }
}
```

---

## 5. 요약 및 주의사항

1.  **N+1 문제:** 연관 관계 조회 시 불필요한 쿼리가 많이 나가는 현상을 주의해야 합니다. (Fetch Join 등으로 해결)
2.  **영속성 컨텍스트:** JPA가 객체의 상태 변화를 감지하여 DB에 반영하는 원리를 이해해야 합니다.
3.  **DB 독립성:** Spring Data JPA를 쓰면 코드는 그대로 두고 설정만 바꿔서 다른 종류의 DB로 쉽게 전환할 수 있습니다.

이제 복잡한 JDBC 코드 대신 JPA로 깔끔하게 데이터베이스를 관리해 보세요!
