# Spring Boot 전역 예외 처리 (Global Exception Handling)

애플리케이션 전반에서 발생하는 예외를 한 곳에서 처리하면 일관된 에러 응답을 제공할 수 있고 비즈니스 로직에 집중할 수 있습니다. 스프링 부트에서 가장 권장되는 `@RestControllerAdvice` 활용법을 소개합니다.

---

## 1. 개요: 개별 처리가 아닌 전역 처리의 필요성

모든 컨트롤러 메소드마다 `try-catch`를 작성하는 것은 번거롭고 중복된 코드를 발생시킵니다. 이를 해결하기 위해 스프링은 **AOP(관점 지향 프로그래밍)** 기술을 활용한 전역 예외 처리 메커니즘을 제공합니다.

---

## 2. 구현 단계

### **① 공통 에러 응답 객체 (Error Response)**
클라이언트에게 전달할 일정한 형식의 에러 메시지 객체를 정의합니다.

```java
@Getter
@Builder
public class ErrorResponse {
    private String code;
    private String message;
    private int status;
}
```

### **② 사용자 정의 예외 (Custom Exception)**
비즈니스 로직에 맞춰 다양한 예외를 구별하기 위해 커스텀 예외를 정의합니다.

```java
public class UserNotFoundException extends RuntimeException {
    public UserNotFoundException(String message) {
        super(message);
    }
}
```

### **③ 전역 예외 처리기 (@RestControllerAdvice)**
모든 컨트롤러에서 발생하는 예외를 가로채어 처리하는 클래스입니다.

```java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(UserNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleUserNotFound(UserNotFoundException ex) {
        ErrorResponse response = ErrorResponse.builder()
                .code("USER_NOT_FOUND")
                .message(ex.getMessage())
                .status(HttpStatus.NOT_FOUND.value())
                .build();
        return new ResponseEntity<>(response, HttpStatus.NOT_FOUND);
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGeneralException(Exception ex) {
        ErrorResponse response = ErrorResponse.builder()
                .code("INTERNAL_SERVER_ERROR")
                .message("서버 내부 오류가 발생했습니다.")
                .status(HttpStatus.INTERNAL_SERVER_ERROR.value())
                .build();
        return new ResponseEntity<>(response, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```

---

## 3. 핵심 장점

1.  **비즈니스 로직 단순화:** 컨트롤러에서 예외 처리 로직을 완전히 제거할 수 있습니다.
2.  **일관된 에러 규격:** 모든 API가 동일한 JSON 에러 형식을 응답하게 됩니다.
3.  **유지보수 용이성:** 새로운 예외가 추가되어도 `GlobalExceptionHandler` 한 곳만 수정하면 됩니다.

---

## 4. 요약

*   `@RestControllerAdvice`를 사용하여 모든 컨트롤러의 예외를 중앙 관리합니다.
*   `@ExceptionHandler`를 통해 특정 예외 타입별로 응답을 정의합니다.
*   클라이언트를 위한 표준 에러 응답 객체를 정의하여 사용합니다.

예외 처리를 깔끔하게 관리하는 것이 견고한 백엔드 시스템 구축의 시작입니다.
