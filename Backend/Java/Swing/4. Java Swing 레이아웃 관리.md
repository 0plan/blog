# Java Swing 레이아웃 관리: 효율적인 화면 배치 방법

자바 스윙(Java Swing)으로 GUI를 개발할 때 가장 어려운 부분 중 하나가 컴포넌트들을 원하는 위치에 배치하는 것입니다. 스윙은 절대 좌표 대신 **레이아웃 매니저(Layout Manager)**를 사용하여 화면 크기 변화에도 유연하게 대응합니다. 

---

## 1. 주요 레이아웃 매니저

스윙에서 가장 기본적이면서 실무에 자주 쓰이는 3가지 레이아웃을 이해해야 합니다.

### **① BorderLayout (기본 레이아웃)**
`JFrame`의 기본 레이아웃으로, 화면을 5개 구역(북, 남, 동, 서, 중앙)으로 나눕니다.
*   **North, South:** 가로로 길게 배치됩니다.
*   **East, West:** 세로로 길게 배치됩니다.
*   **Center:** 나머지 전체 공간을 차지하며, 창 크기를 조절할 때 함께 늘어납니다.

```java
JPanel panel = new JPanel(new BorderLayout());
panel.add(new JButton("메뉴"), BorderLayout.NORTH);
panel.add(new JButton("로그"), BorderLayout.SOUTH);
panel.add(new JTextArea(), BorderLayout.CENTER);
```

### **② FlowLayout**
컴포넌트를 한 줄로 차례대로 배치합니다. 줄이 꽉 차면 자동으로 다음 줄로 넘어갑니다. 버튼들을 나란히 배치할 때 주로 쓰입니다.

```java
JPanel buttonPanel = new JPanel(new FlowLayout(FlowLayout.RIGHT)); // 우측 정렬
buttonPanel.add(new JButton("확인"));
buttonPanel.add(new JButton("취소"));
```

### **③ GridLayout**
화면을 일정한 크기의 격자(바둑판) 형태로 나눕니다. 모든 컴포넌트의 크기가 동일하게 유지됩니다. 계산기 같은 UI를 만들 때 최적입니다.

```java
JPanel gridPanel = new JPanel(new GridLayout(3, 2)); // 3행 2열
gridPanel.add(new JButton("1"));
gridPanel.add(new JButton("2"));
// ... 계속 추가
```

---

## 2. 복합 레이아웃 전략 (Nested Panels)

하나의 레이아웃만으로는 복잡한 화면을 만들 수 없습니다. 가장 중요한 기법은 **`JPanel`을 중첩해서 사용**하는 것입니다.

*   **Step 1:** 메인 프레임에는 `BorderLayout`을 사용합니다.
*   **Step 2:** 남쪽(`SOUTH`) 구역에 `FlowLayout`을 가진 `JPanel`을 넣어 버튼들을 배치합니다.
*   **Step 3:** 중앙(`CENTER`) 구역에 `GridLayout`을 가진 `JPanel`을 넣어 입력 폼을 배치합니다.

---

## 3. 레이아웃 관련 실무 팁

1.  **Pack() 메서드:** `frame.setSize()` 대신 `frame.pack()`을 호출하면, 컴포넌트들의 권장 크기(Preferred Size)에 맞춰 창 크기가 자동으로 조절됩니다.
2.  **Margin과 Padding:** 컴포넌트 사이에 여백을 주고 싶다면 `EmptyBorder`를 활용하거나, 레이아웃 매니저 생성 시 간격(hgap, vgap)을 설정하세요.
3.  **컴포넌트 크기 고정:** 레이아웃 매니저가 컴포넌트 크기를 강제로 조절하므로, `setPreferredSize()`를 사용하여 힌트를 주는 것이 좋습니다.

---

## 4. 요약

*   화면의 큰 틀은 **`BorderLayout`**으로 잡습니다.
*   세밀한 배치는 **`JPanel`을 중첩**하여 각각 다른 레이아웃을 적용합니다.
*   절대 좌표(`setLayout(null)`)는 창 크기 조절 시 UI가 깨지므로 가급적 피해야 합니다.

적절한 레이아웃 선택은 유지보수가 쉬운 깔끔한 UI의 시작입니다.
