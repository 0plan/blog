# Java Swing 데이터 연동: UI와 데이터 모델 연결하기

자바 스윙(Java Swing) 개발에서 가장 중요한 원칙 중 하나는 **화면(View)과 데이터(Model)의 분리**입니다. 단순히 컴포넌트에 직접 데이터를 쓰는 방식에서 벗어나, 데이터 모델을 활용해 UI를 동기화하는 방법을 정리합니다.

---

## 1. 텍스트 필드 데이터 가져오기 및 감시

사용자가 입력하는 데이터를 실시간으로 감지하거나 가져오는 방법입니다.

*   **가져오기:** `textField.getText()`를 사용하여 간단히 가져옵니다.
*   **실시간 감시:** `DocumentListener`를 사용하면 글자가 입력될 때마다 이벤트를 받을 수 있습니다.

```java
JTextField textField = new JTextField();
textField.getDocument().addDocumentListener(new DocumentListener() {
    public void changedUpdate(DocumentEvent e) { update(); }
    public void removeUpdate(DocumentEvent e) { update(); }
    public void insertUpdate(DocumentEvent e) { update(); }

    public void update() {
        System.out.println("입력 내용: " + textField.getText());
    }
});
```

## 2. 테이블 데이터 연동 (JTable과 TableModel)

스윙에서 가장 강력한 기능인 **`JTable`**은 데이터를 직접 넣지 않고 반드시 **`TableModel`**을 통해 연동해야 합니다.

### **① 기본 방식 (DefaultTableModel)**
가장 쉽고 빠르게 테이블 데이터를 추가/삭제할 때 사용합니다.

```java
String[] columns = {"ID", "이름", "이메일"};
DefaultTableModel model = new DefaultTableModel(columns, 0);
JTable table = new JTable(model);

// 데이터 추가
model.addRow(new Object[]{"1", "홍길동", "hong@example.com"});

// 데이터 삭제 (첫 번째 행)
model.removeRow(0);
```

### **② 고급 방식 (AbstractTableModel)**
실제 비즈니스 객체(`List<User>` 등)와 테이블을 직접 연결하고 싶을 때 상속받아 구현합니다. 데이터의 원본이 바뀌면 화면이 자동으로 갱신되도록 제어할 수 있습니다.

---

## 3. UI 업데이트 시 주의사항 (데이터 무결성)

데이터베이스나 네트워크에서 가져온 데이터를 UI에 뿌려줄 때는 항상 **EDT(Event Dispatch Thread)**를 거쳐야 함을 잊지 마세요.

```java
// 별도 스레드에서 데이터를 가져온 후
List<User> userList = db.findAllUsers();

// UI 업데이트는 반드시 SwingUtilities를 사용
SwingUtilities.invokeLater(() -> {
    for (User user : userList) {
        model.addRow(new Object[]{user.getId(), user.getName()});
    }
});
```

---

## 4. 실무 패턴: POJO와 UI 싱크 맞추기

단순히 `getText()`를 쓰는 것보다, 별도의 **데이터 객체(POJO)**를 정의하고 컴포넌트의 값을 객체에 담아 관리하는 것이 유지보수에 유리합니다.

1.  **화면 → 객체:** '저장' 버튼 클릭 시 모든 텍스트 필드의 값을 `User` 객체에 세팅합니다.
2.  **객체 → 화면:** '수정' 버튼 클릭 시 `User` 객체의 값을 각 텍스트 필드에 `setText()` 해줍니다.

---

## 5. 요약

*   간단한 입력은 **`getText()` / `setText()`**로 충분합니다.
*   복잡한 목록(Table, List)은 반드시 **Model 인터페이스**를 사용하세요.
*   대량의 데이터를 UI에 연동할 때는 **스레드 분리**가 필수입니다.

데이터와 화면을 분리하는 습관은 대규모 스윙 애플리케이션 개발의 핵심입니다.
