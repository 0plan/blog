# Java Swing 이벤트 처리: 사용자와의 상호작용 이해하기

자바 스윙(Java Swing) 애플리케이션에서 버튼 클릭, 키보드 입력 등 사용자의 동작을 처리하는 핵심 메커니즘은 **이벤트 리스너(Event Listener)** 모델입니다. 2026년 기준 현대적인 자바 문법(람다식 등)을 활용한 효율적인 이벤트 처리 방법을 정리합니다.

---

## 1. 이벤트 리스너 모델의 구조

스윙의 이벤트 처리는 **'발생지(Source)'**, **'이벤트 객체(Event Object)'**, **'리스너(Listener)'**의 세 요소로 구성됩니다.

1.  **Source:** 이벤트가 발생하는 컴포넌트 (예: `JButton`)
2.  **Event Object:** 발생한 이벤트에 대한 정보를 담은 객체 (예: `ActionEvent`)
3.  **Listener:** 이벤트를 감시하고 있다가 발생 시 로직을 수행하는 인터페이스 (예: `ActionListener`)

---

## 2. 주요 이벤트 처리 방법

### **① 버튼 클릭 처리 (ActionListener)**
가장 많이 쓰이는 방식입니다. 과거에는 익명 클래스를 썼지만, 현재는 **람다식**을 사용하여 간결하게 작성합니다.

```java
JButton button = new JButton("클릭하세요");

// 람다식을 이용한 이벤트 처리
button.addActionListener(e -> {
    System.out.println("버튼이 클릭되었습니다!");
    JOptionPane.showMessageDialog(null, "안녕하세요!");
});
```

### **② 마우스 및 키보드 이벤트**
특정 컴포넌트 위로 마우스를 올리거나 키보드 키를 누르는 동작을 감지합니다.

*   **MouseListener:** `mouseClicked`, `mouseEntered`, `mouseExited` 등 처리.
*   **KeyListener:** `keyPressed`, `keyReleased` 등 처리.

> **Tip:** 모든 메서드를 오버라이드하기 번거롭다면 `MouseAdapter`나 `KeyAdapter`를 상속받아 필요한 메서드만 구현할 수 있습니다.

---

## 3. 중요: 이벤트 처리 시 주의사항 (EDT와 스레드)

지난 글에서 강조했듯, 스윙은 싱글 스레드 모델입니다. 이벤트 리스너 내부의 코드는 **EDT(Event Dispatch Thread)**에서 실행됩니다.

*   **금지 사항:** 리스너 내부에서 네트워크 통신, 대용량 파일 읽기 등 '무거운 작업'을 직접 수행하면 안 됩니다. 작업이 끝날 때까지 화면이 멈추게 됩니다.
*   **해결 방법:** 시간이 걸리는 작업은 `SwingWorker` 클래스를 사용하거나 별도의 스레드에서 처리한 뒤, UI 업데이트만 다시 `SwingUtilities.invokeLater()`로 넘겨야 합니다.

```java
button.addActionListener(e -> {
    new Thread(() -> {
        // 1. 무거운 작업 수행 (예: DB 조회)
        String data = fetchDataFromDB(); 
        
        // 2. UI 업데이트는 다시 EDT에서 실행
        SwingUtilities.invokeLater(() -> {
            label.setText(data);
        });
    }).start();
});
```

---

## 4. 요약

*   간단한 액션은 **람다식**과 `ActionListener`로 처리합니다.
*   복잡한 마우스/키보드 동작은 **Adapter 클래스**를 활용해 코드를 간결하게 유지합니다.
*   사용자 경험을 위해 **무거운 작업은 반드시 별도 스레드**로 분리합니다.

이벤트 처리를 마스터하면 정적인 UI에 생명력을 불어넣을 수 있습니다.
