# Java Swing이란 무엇인가?

자바(Java)는 서버 사이드뿐만 아니라 데스크톱 애플리케이션 개발을 위해서도 강력한 도구들을 제공해 왔습니다. 그중에서도 **Swing**은 가장 오랫동안 자바 GUI(Graphical User Interface) 개발의 중심에 있었던 라이브러리입니다. 2026년 현재는 JavaFX와 같은 현대적인 대안도 있지만, 여전히 많은 기업용 사내 툴과 기존 시스템의 유지보수에서 널리 사용되고 있습니다.

---

## 1. Java Swing의 핵심 개념

Swing은 **JFC(Java Foundation Classes)**의 일부로, 순수 자바 코드로 작성된 GUI 툴킷입니다. 이전의 AWT(Abstract Window Toolkit)가 각 운영체제(OS)의 네이티브 위젯을 그대로 가져다 쓴 것과 달리, Swing은 자바가 자체적으로 컴포넌트를 그리는 **'경량(Lightweight)'** 방식을 취합니다.

### **핵심 특징**
1. **플랫폼 독립성:** 자바 코드만으로 컴포넌트를 직접 렌더링하므로, 어떤 운영체제에서도 일관된 모습을 보여줍니다.
2. **PLAF (Pluggable Look and Feel):** 코드 한 줄로 윈도우 스타일, 맥 스타일, 혹은 자바 고유의 'Metal' 스타일 등 프로그램의 전체적인 테마를 손쉽게 바꿀 수 있습니다.
3. **MVC 아키텍처:** 컴포넌트의 데이터(Model), 화면 표시(View), 제어 로직(Controller)이 분리되어 있어 구조적인 개발이 가능합니다.
4. **풍부한 위젯:** 버튼(JButton), 텍스트 필드(JTextField), 표(JTable), 트리(JTree) 등 매우 다양하고 고도화된 컴포넌트들을 기본 제공합니다.

---

## 2. Swing의 내부 구조: 이벤트 기반 모델

Swing은 **싱글 스레드 모델**로 동작합니다. 모든 UI 업데이트와 이벤트 처리는 **EDT(Event Dispatch Thread)**라는 전용 스레드에서만 이루어져야 합니다.

*   사용자가 버튼을 클릭하면, 클릭 이벤트가 발생하고 EDT에 있는 큐에 쌓입니다.
*   EDT는 하나씩 이벤트를 꺼내어 처리하고 화면을 다시 그립니다.
*   **주의:** 시간이 오래 걸리는 작업(네트워크 통신, 대용량 DB 조회 등)을 EDT에서 직접 수행하면 프로그램 화면이 멈추는(Freezing) 현상이 발생하므로, 별도의 스레드(`SwingWorker` 등)에서 처리해야 합니다.

---

## 3. Java Swing의 장단점

### **장점**
*   **성숙함:** 20년 넘게 사용되어 왔기에 수많은 라이브러리와 예제 코드가 존재합니다.
*   **간결함:** 외부 라이브러리 설치 없이 JDK만 있으면 바로 시작할 수 있습니다.
*   **강력한 표(Table) 기능:** `JTable`은 데이터 시각화와 편집 기능이 매우 강력하여 관리자용 툴 제작에 최적입니다.

### **단점**
*   **디자인의 한계:** 최신 웹이나 모바일 앱 같은 화려한 UI를 CSS처럼 편하게 적용하기는 어렵습니다.
*   **성능:** 네이티브 위젯을 사용하는 라이브러리에 비해 아주 미세하게 반응 속도가 느릴 수 있습니다.

---

## 4. 결론: 2026년에 Swing을 배워야 할까?

이미 구축된 대규모 엔터프라이즈 시스템의 관리 도구가 Swing으로 되어 있는 경우가 많아, 백엔드 개발자에게는 여전히 유용한 기술입니다. 특히 **JetBrains의 IntelliJ IDEA**가 Swing으로 만들어졌다는 사실은 이 기술이 여전히 얼마나 강력하고 복잡한 애플리케이션을 지탱할 수 있는지를 보여주는 좋은 예시입니다.

데스크톱 앱 개발의 기초를 탄탄히 다지고 싶거나, 빠르게 내부용 툴을 만들어야 한다면 Swing은 여전히 훌륭한 선택입니다.
