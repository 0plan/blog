# 데이터베이스 종류 및 특징 비교: SQL vs NoSQL (2026년 기준)

현대 소프트웨어 아키텍처에서 데이터베이스(Database)는 서비스의 성능과 확장성을 결정짓는 핵심 요소입니다. 단순히 "데이터를 저장하는 곳"을 넘어, 프로젝트의 성격에 따라 최적의 도구를 선택하는 역량이 중요합니다. 실무에서 가장 많이 쓰이는 주요 데이터베이스들의 특징과 차이점을 정리해 보겠습니다.

---

## 1. 관계형 데이터베이스 (SQL, RDBMS)

**특징:** 데이터를 테이블 형태(행과 열)로 저장하며, 정해진 스키마(Schema)에 따라 데이터의 관계를 엄격히 관리합니다.

### **MySQL: 가장 대중적인 오픈소스 DB**
*   **장점:** 설치와 사용이 매우 간편하며, 웹 서비스 개발에서 가장 널리 쓰여 커뮤니티와 자료가 풍부합니다.
*   **용도:** 대부분의 일반적인 웹/앱 서비스, 소규모~대규모 프로젝트의 메인 DB.

### **PostgreSQL: 강력한 확장성과 안정성**
*   **장점:** 복잡한 쿼리 처리 능력이 뛰어나며, JSONB(비정형 데이터) 지원 등 NoSQL의 장점을 일부 흡수했습니다. 오픈소스 중 가장 '표준 SQL'에 가깝습니다.
*   **용도:** 복잡한 비즈니스 로직, 데이터 무결성이 중요한 금융 서비스, 지리 정보 시스템(GIS).

### **Oracle: 엔터프라이즈의 표준**
*   **장점:** 대용량 데이터 처리와 고가용성(High Availability) 측면에서 압도적인 성능과 안정성을 제공합니다.
*   **용도:** 대기업, 금융권, 공공기관 등 안정성이 절대적인 대규모 핵심 시스템.

---

## 2. 비관계형 데이터베이스 (NoSQL)

**특징:** 고정된 스키마 없이 데이터를 문서(Document), 키-값(Key-Value) 등 유연한 형태로 저장하며, 수평적 확장(Scale-out)이 쉽습니다.

### **MongoDB: 문서 기반(Document) DB**
*   **장점:** JSON 형태의 문서를 그대로 저장하므로 데이터 구조가 유연하고 개발 속도가 매우 빠릅니다.
*   **용도:** 로그 데이터, 빈번한 스키마 변경이 필요한 초기 스타트업, 콘텐츠 관리 시스템.

### **Redis: 인메모리(In-memory) DB**
*   **장점:** 모든 데이터를 메모리에 저장하여 응답 속도가 '나노초(ns)' 단위로 매우 빠릅니다.
*   **용도:** 캐시(Cache), 실시간 랭킹 시스템, 세션 관리, 메시지 브로커.

### **Cassandra: 광범위한 쓰기 성능**
*   **장점:** 수많은 서버에 데이터를 분산 저장하여 쓰기(Write) 성능이 매우 뛰어나고 장애 내성이 강합니다.
*   **용도:** 넷플릭스 등 전 세계 규모의 서비스, 방대한 양의 트랜잭션 데이터 로그.

---

## 3. 2026년의 새로운 트렌드: Vector Database

최근 생성형 AI(Generative AI)의 폭발적인 성장으로 **벡터 데이터베이스(Vector DB)**가 필수 기술로 자리 잡았습니다.

*   **특징:** 데이터를 숫자의 배열인 '벡터'로 변환하여 저장하며, 의미적 유사도 검색(Similarity Search)을 수행합니다.
*   **주요 도구:** Pinecone, Milvus, Chroma, Weaviate 등.
*   **용도:** 챗봇의 지식 베이스(RAG), 개인화 추천 시스템, 이미지/영상 검색 서비스.

---

## 4. 데이터베이스 비교 및 선택 가이드

| 비교 항목 | SQL (관계형) | NoSQL (비관계형) |
| :--- | :--- | :--- |
| **데이터 구조** | 엄격한 테이블 구조 (스키마) | 유연한 문서/키-값 구조 |
| **확장성** | 수직 확장 (Scale-up) 위주 | 수평 확장 (Scale-out) 유리 |
| **트랜잭션** | ACID 지원 (강력한 일관성) | BASE (유연한 일관성) |
| **적합한 상황** | 관계가 복잡하고 일관성이 중요함 | 데이터가 방대하고 확장이 중요함 |

### **무엇을 선택해야 할까?**
1. **표준적인 웹 서비스 개발:** **MySQL** 또는 **PostgreSQL**로 시작하세요.
2. **실시간 검색이나 빠른 캐싱이 필요:** **Redis**를 보조 DB로 도입하세요.
3. **데이터의 형태가 정해져 있지 않고 빠르게 성장해야 함:** **MongoDB**를 고려하세요.
4. **AI 모델의 지식 저장소가 필요:** **Vector DB**를 찾아보세요.

### **결론**
현대적인 아키텍처는 하나의 데이터베이스만 쓰지 않습니다. 메인 데이터는 **SQL**에 담고, 성능 향상을 위한 캐시에는 **Redis**를, 대규모 로그나 비정형 데이터에는 **NoSQL**을 혼합하여 사용하는 **폴리글랏 퍼시스턴스(Polyglot Persistence)**가 권장되는 전략입니다.
