# 데이터베이스 실전 사용법 및 시나리오별 예제

데이터베이스 종류를 아는 것보다 중요한 것은 **"어떻게 효율적으로 데이터를 다루는가(CRUD)"**입니다. 가장 많이 쓰이는 SQL과 NoSQL의 핵심 사용법을 실무 시나리오와 함께 정리합니다.

---

## 1. 관계형 데이터베이스 (SQL - MySQL/PostgreSQL)

### **시나리오: 쇼핑몰 주문 및 회원 관리**

#### **① 데이터 생성 (INSERT)**
```sql
INSERT INTO users (name, email) VALUES ('홍길동', 'hong@example.com');
```

#### **② 복잡한 조건 조회 (SELECT + JOIN)**
```sql
SELECT u.name, o.order_date, o.total_price
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE u.name = '홍길동'
ORDER BY o.order_date DESC;
```

#### **③ 데이터 수정 (UPDATE)**
```sql
UPDATE users SET email = 'new_hong@example.com' WHERE id = 1;
```

#### **④ 데이터 삭제 (DELETE) - 주의사항 포함**
*   **Hard Delete (물리 삭제):** 데이터를 DB에서 완전히 지웁니다.
    ```sql
    DELETE FROM users WHERE id = 1; -- WHERE절이 없으면 모든 데이터가 삭제되니 주의!
    ```
*   **Soft Delete (논리 삭제 - 실무 권장):** 실제로 지우지 않고 '삭제 여부' 상태만 변경합니다. 데이터 복구가 가능하고 통계 분석에 유리합니다.
    ```sql
    UPDATE users SET is_deleted = true, deleted_at = NOW() WHERE id = 1;
    ```

---

## 2. 문서 기반 데이터베이스 (NoSQL - MongoDB)

### **시나리오: 블로그 포스트 관리**

#### **① 데이터 생성 및 수정**
```javascript
db.posts.insertOne({ title: "안녕, 몽고DB!", content: "유연한 구조" });
db.posts.updateOne({ title: "안녕, 몽고DB!" }, { $set: { content: "수정된 내용" } });
```

#### **② 데이터 삭제 (Delete)**
*   **단건 삭제:** 조건에 맞는 첫 번째 문서만 삭제합니다.
    ```javascript
    db.posts.deleteOne({ _id: ObjectId("60f...") });
    ```
*   **다건 삭제:** 조건에 맞는 모든 문서를 삭제합니다.
    ```javascript
    db.posts.deleteMany({ status: "draft" }); // 초안 상태인 모든 글 삭제
    ```

---

## 3. 키-값 데이터베이스 (NoSQL - Redis)

### **시나리오: 세션 및 캐시 관리**

#### **① 값 저장 및 삭제**
*   **삭제 (DEL):** 키를 즉시 삭제합니다.
    ```bash
    DEL session:user:1
    ```
*   **자동 삭제 (Expire):** 특정 시간 뒤에 자동으로 사라지게 합니다.
    ```bash
    EXPIRE session:user:1 3600 # 1시간 뒤 삭제
    ```

---

## 4. 삭제 작업 시 실무 가이드 (Best Practice)

1.  **WHERE절 확인 또 확인:** `UPDATE`나 `DELETE`를 실행하기 전에는 반드시 같은 조건으로 `SELECT`를 먼저 해보세요. 내가 지우려는 데이터가 맞는지 확인하는 습관이 대형 사고를 막습니다.
2.  **가급적 Soft Delete 사용:** 회원 탈퇴나 주문 취소 같은 데이터는 나중에 분쟁 해결이나 데이터 분석을 위해 `is_deleted` 같은 컬럼을 활용해 보관하는 것이 좋습니다.
3.  **백업 및 로그:** 중요한 데이터를 지울 때는 삭제 전 로그를 남기거나 배포 전 백업본이 있는지 확인하세요.

### **결론**
생성(C), 조회(R), 수정(U)도 중요하지만, **삭제(D)는 가장 위험하고 신중해야 하는 작업**입니다. 각 DB의 특성에 맞는 삭제 전략을 세워 데이터의 안전성을 지키세요!
