# Linux 핵심 명령어 및 서버 운영 기초: 개발자를 위한 터미널 가이드

우리가 배포하는 대부분의 서버(AWS, Oracle Cloud, 홈서버 등)는 리눅스(Linux) 운영체제에서 동작합니다. 마우스가 없는 검은 화면(터미널)에서 서버를 자유롭게 다루기 위한 필수 지식들을 정리합니다.

---

## 1. 파일 및 디렉토리 조작

*   **`ls -al`**: 현재 디렉토리의 모든 파일 목록을 자세히(권한, 소유자 포함) 보여줍니다.
*   **`cd <경로>`**: 지정한 경로로 이동합니다. (`cd ..`은 상위 디렉토리)
*   **`mkdir <이름>`**: 새로운 디렉토리를 생성합니다.
*   **`rm -rf <이름>`**: 파일이나 디렉토리를 강제로 삭제합니다. (매우 주의해서 사용!)
*   **`cp` / `mv`**: 파일을 복사하거나 이동(이름 변경)합니다.

---

## 2. 권한 관리 (Permissions)

리눅스는 다중 사용자 시스템이므로 권한 관리가 매우 중요합니다.
*   **`sudo <명령어>`**: 관리자(Root) 권한으로 명령어를 실행합니다.
*   **`chmod <권한> <파일>`**: 파일의 읽기/쓰기/실행 권한을 변경합니다. (예: `chmod 755 script.sh`)
*   **`chown <사용자>:<그룹> <파일>`**: 파일의 소유자를 변경합니다.

---

## 3. 프로세스 및 시스템 모니터링

서버가 느려지거나 멈췄을 때 상황을 파악하는 명령어들입니다.
*   **`top` / `htop`**: CPU와 메모리 사용량을 실시간으로 확인합니다.
*   **`ps -ef | grep <이름>`**: 실행 중인 특정 프로세스를 찾습니다.
*   **`kill -9 <PID>`**: 특정 프로세스를 강제로 종료합니다.
*   **`df -h`**: 디스크 용량 상태를 확인합니다.

---

## 4. 네트워크 및 로그 확인

*   **`netstat -ntlp`**: 현재 서버에서 열려 있는 포트 번호들을 확인합니다.
*   **`curl -I <URL>`**: 특정 URL이 정상적으로 응답하는지 확인합니다.
*   **`tail -f <로그파일>`**: 로그 파일의 뒷부분을 실시간으로 계속 보여줍니다. (디버깅 시 필수!)
*   **`grep "ERROR" <로그파일>`**: 로그 파일에서 특정 단어를 검색합니다.

---

## 5. 서버 운영 팁

1.  **SSH 접속:** `ssh -i <키파일> <사용자>@<IP주소>`를 통해 원격으로 서버에 접속합니다.
2.  **Tab 자동완성:** 긴 경로나 파일명을 입력할 때 `Tab` 키를 누르면 자동으로 완성됩니다.
3.  **히스토리 검색:** `Ctrl + R`을 누르고 이전에 입력했던 명령어를 검색할 수 있습니다.

리눅스 터미널은 처음엔 낯설지만, 익숙해지면 마우스보다 훨씬 빠르고 강력한 도구가 됩니다. 하나씩 직접 실행해보며 익혀보세요!
