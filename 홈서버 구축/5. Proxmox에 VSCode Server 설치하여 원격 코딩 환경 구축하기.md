# Proxmox에 VSCode Server(Code-Server) 설치하여 원격 코딩 환경 구축하기

Proxmox 홈서버를 구축한 뒤 가장 먼저 하고 싶은 일 중 하나는 바로 **어디서든 접속 가능한 코딩 환경**을 만드는 것입니다. 이번 포스트에서는 Proxmox 위에 VSCode Server(Code-Server)를 설치하고, 외부에서도 안전하게 접속할 수 있도록 설정하는 방법을 알아보겠습니다.

---

## 1. 왜 Proxmox에 VSCode Server를 설치하나요?

- **디바이스 독립성**: 태블릿, 노트북, PC 어디서든 브라우저만 있으면 동일한 코딩 환경에 접속할 수 있습니다.
- **서버 리소스 활용**: 저사양 노트북에서도 서버의 강력한 CPU와 RAM을 사용하여 빌드 및 테스트를 수행할 수 있습니다.
- **Proxmox의 유연성**: LXC 컨테이너나 VM으로 가볍게 분리하여 관리할 수 있어 시스템이 깔끔합니다.

---

## 2. 사전 준비사항

- **Proxmox 서버**가 설치되어 있어야 합니다. (아직 설치 전이라면 [Proxmox 홈서버 만들기](./1.%20Proxmox를%20이용한%20홈서버%20만들기.md) 참고)
- **Ubuntu LXC 컨테이너** 또는 **VM** (이 가이드에서는 가벼운 LXC 컨테이너 기준입니다.)
- **Docker** 설치 (권장)

---

## 3. VSCode Server (Code-Server) 설치하기

가장 관리하기 쉬운 **Docker Compose** 방식을 추천합니다.

### 3.1 Docker Compose 파일 작성
LXC 내 적절한 위치에 `docker-compose.yml` 파일을 생성합니다.

```yaml
version: "3"
services:
  code-server:
    image: codercom/code-server:latest
    container_name: code-server
    environment:
      - PASSWORD=your_secure_password  # 접속 비밀번호 설정
      - TZ=Asia/Seoul
    volumes:
      - ./config:/home/coder/.config
      - ./project:/home/coder/project  # 코드가 저장될 폴더
    ports:
      - 8080:8080
    restart: unless-stopped
```

### 3.2 컨테이너 실행
```bash
docker compose up -d
```
이제 내부 네트워크에서 `http://[LXC-IP]:8080`으로 접속하여 비밀번호를 입력하면 VSCode 화면을 볼 수 있습니다.

---

## 4. 외부 접속 설정 (Nginx Proxy Manager)

집 밖에서도 접속하려면 도메인 연결과 SSL 보안 설정이 필수입니다.

1. **Nginx Proxy Manager(NPM)** 관리 페이지에 접속합니다. ([NPM 설치 방법 바로가기](./2.%20Nginx%20Proxy%20Manager(NPM)%20설치하기.md))
2. **Proxy Host**를 추가합니다.
   - **Domain Names**: `vscode.yourdomain.com`
   - **Scheme**: `http`
   - **Forward Hostname / IP**: VSCode Server가 설치된 LXC의 IP
   - **Forward Port**: `8080`
   - **Websockets Support**: **반드시 체크** (VSCode의 터미널 및 실시간 기능을 위해 필수입니다.)
3. **SSL** 탭에서 Let's Encrypt 인증서를 발급받아 적용합니다.

---

## 5. 트러블슈팅 및 팁

### 터미널이 작동하지 않나요?
NPM 설정에서 **Websockets Support**를 활성화했는지 다시 확인하세요. VSCode와 브라우저 간의 통신은 웹소켓을 기반으로 합니다.

### 권한 문제 (Permission Denied)
Docker 볼륨으로 연결된 폴더의 권한이 `coder` 유자(UID 1000)에게 있는지 확인하세요.
```bash
sudo chown -R 1000:1000 ./project
```
